📋 Talking points — Optimization

  • Load matching is the primary tool for reducing empty miles. The concept is simple: instead of a truck returning empty after a delivery, a dispatcher (or algorithm) finds a nearby shipper who needs freight moved in that direction.

What are empty miles?

Empty miles (also called "deadhead miles") refer to the distance a truck travels without carrying any freight. This happens when a driver delivers a load and then drives back to base — or to the next pickup — without a return load. In the trucking industry, the driver and truck still incur full operating costs (fuel, wear, driver wages) during those empty miles, but the carrier earns zero revenue. It is one of the most significant sources of inefficiency in freight logistics.

Can empty miles be eliminated?

1
No — not entirely. Some empty miles are structurally unavoidable. When a truck delivers to a rural or one-directional market, there may simply be no return freight available. Regulatory limits (Hours of Service rules) also sometimes prevent waiting for a backhaul load.
2
Yes — significantly reducible. Industry research shows that systematic load matching can reduce empty miles by 40–60% in dense freight markets like NY/NJ. Best-in-class carriers achieve empty rates below 15%.
3
The NY/NJ corridor is ideal for optimization. High freight density in both directions — imports via Port Newark, outbound consumer goods and manufacturing — means backhaul opportunities are abundant. The problem is matching, not supply.

Proposed solution — load matching

✓ Phase 1 (Month 1–2): Subscribe to a load board (DAT, Truckstop.com) — cost ~$150/month. Dispatchers manually identify backhaul loads for the worst-performing lanes (Port Newark → Bethlehem PA, Meadowlands → Albany NY). Target: 40% match rate.
✓ Phase 2 (Month 3–4): Implement dispatch software with route optimization (e.g. Samsara, KeepTruckin). Automate backhaul suggestions. Target: 60% match rate.
✓ Phase 3 (Month 5–6): Integrate with freight brokerage APIs for real-time load matching. Driver app notifications for available backhauls. Target: 75–80% match rate.

Conclusion

Empty miles cannot be eliminated from freight operations, but they can be dramatically reduced through systematic load matching. For the NY/NJ/CT/PA corridor — one of the densest freight markets in North America — a realistic target of 55–60% load match rate is achievable within six months, reducing J. Hammerhead Trucking's empty rate from 41% to approximately 16–24%. At $2.50 per mile in operating costs, this translates to annual savings of $17,220–$22,980 for a 6-driver independent fleet. Scaled to a larger operation, the impact is proportionally greater. The technology exists, the freight density supports it, and the financial case is clear.

Project journey

How this project was scoped, built and developed — step by step.

1
Scoping the project

The project started with a 3-hour build goal and the topic of "empty miles." We first narrowed down what was needed: a web app dashboard UI, built with JavaScript, focused on trucking and freight — specifically unloaded truck legs. We confirmed 3 hours was feasible for a well-scoped dashboard.

2
First working prototype

An initial interactive dashboard was built with mock data covering the core empty miles concept — KPI cards, a stacked bar chart, a donut chart by region, and a trip table with filters for date range, driver, and region. This gave a working foundation to react to.

3
Making the data realistic

The placeholder data was replaced with 6 named drivers (Marcus Webb, Sandra Kowalski, Darnell Pruitt, Carolyn Hutchins, Ray Thibodeau, Yvette Salgado), US-format phone numbers using the 555 prefix convention, and actual highway mileages between real US city pairs. A driver roster panel was added to the dashboard.

4
Defining the problem and solution

The key question was asked: can empty miles actually be eliminated? The honest answer — no, not entirely, but yes, significantly reducible. The US industry average sits around 35% empty miles, and well-optimized fleets get it to 15–20% using load matching, route optimization, and predictive dispatch. This became the intellectual backbone of the project.

5
Localizing to NY/NJ

Rather than keeping it generic, the focus was narrowed to the NY/NJ tri-state corridor — one of the densest freight markets in the US, with real anchor hubs: Port Newark, Hunts Point (Bronx), Meadowlands, Edison NJ, and JFK Cargo. This made the project far more credible and grounded for a class presentation.

6
Building the three-tab app

Everything was rebuilt around the NY/NJ market with three tabs forming a clear narrative arc: Dashboard (here's the problem), Route Map (here's where it's happening), and Optimization (here's the solution). The optimization tab included a live slider to simulate different load match rates and see real-time cost savings — the most compelling part of the presentation.

7
Choosing the final format

The best format for a novice was determined to be a single HTML file — no installs, no frameworks, just double-click and it opens in any browser. Simple to share, simple to print, works offline. A Word document would have lost all the interactive charts and maps.

8
Final comprehensive build — delivered

The complete six-tab standalone HTML application was delivered including: the Dashboard with full trip log, Driver Profiles with individual stats and a comparison chart, the interactive Route Map with clickable routes, the Optimization simulator with matched load pairs and a 6-month projection, a Summary Report designed to be printed, and this Project Journey page. Talking points for each tab were embedded directly so you always know what to say during your presentation.

The through-line: Moving from a vague problem ("empty miles are bad") to a specific, defensible, locally-grounded solution — load matching in the NY/NJ/CT/PA corridor can reduce J. Hammerhead Trucking's empty miles from 41% to under 17%, saving an estimated $17,220–$22,980 per year for a 6-driver independent fleet. That is a complete class project argument, backed by a working application.

📋 Talking points — Predictive positioning

  • Predictive positioning moves the solution from reactive to proactive. Instead of finding a backhaul after a driver has already delivered empty, the system analyzes historical trip patterns and predicts where empty legs are likely to occur before they happen.
  • Using each driver's last 30 days of trip data, the system identifies which lanes consistently generate empty return legs, which days of the week are highest risk, and which drivers need attention first.
  • Dispatchers receive a 7-day forward risk calendar — flagging high-risk days in red so they can pre-book return loads in advance rather than scrambling at the last minute.
  • This is the same principle used by companies like Uber Freight and Convoy at scale — the difference is that even a small fleet can benefit from basic pattern recognition on their own historical data.
